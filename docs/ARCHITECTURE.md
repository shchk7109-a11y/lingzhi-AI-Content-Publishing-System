# 灵芝水铺发布系统 — 架构说明

## 系统架构

本系统采用 Electron 桌面应用架构，主要分为以下层次：

### 1. 主进程层 (Main Process)
- 窗口管理、IPC通信、本地文件服务
- 核心业务逻辑运行在主进程中

### 2. 渲染进程层 (Renderer Process)
- React 18 + Ant Design 5 构建的用户界面
- Zustand 状态管理
- 通过 IPC 与主进程通信

### 3. 核心引擎层 (Core)
- 内容组织器：Excel解析和媒体文件匹配
- 智能匹配引擎：基于画像的内容-账号匹配
- 自动化发布：通过 puppeteer-core 连接 Bit浏览器
- 任务调度：令牌桶 + 时间窗双重调度

### 4. 数据层 (Database)
- better-sqlite3 本地数据库
- 7张核心业务表
- 自动迁移机制

## 数据流

1. Excel导入 → 内容池入库
2. 智能匹配 → 生成匹配记录
3. 确认匹配 → 创建发布任务
4. 任务调度 → 窗口池执行 → 平台发布
5. 结果回写 → 状态更新
